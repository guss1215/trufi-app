# gtfs-bolivia-cochabamba

Generates the Cochabamba GTFS feed using [`trufi-association/trufi-gtfs-builder`](https://github.com/trufi-association/trufi-gtfs-builder) as a Node library — no fork, no submodule. The pinned version lives in [`package.json`](package.json).

## Run

```bash
npm install
npm start
```

Output: `./out/cochabamba.gtfs.zip` (committed). Other files under `out/` are also written but only the zip is the canonical artifact.

## Input

Reads `../pbf-bolivia-cochabamba/out/cochabamba.osm.pbf` (generated by the sibling tool). If it isn't there, run that tool first.

The Overpass mode is also wired up but currently broken upstream (the builder calls `www.overpass-api.de` which returns 504); use the PBF.

## Cochabamba tweaks

City-specific values live in [`index.ts`](index.ts): bounding box, `transformTypes`, `skipRoute` IDs, `fakeStops` IDs, agency / calendar / headway / vehicle speed / feed metadata. Edit there.

## Pin

`package.json` pins `trufi-gtfs-builder` to tag `v2.11.0` via a GitHub git URL. Bump that ref to upgrade.
