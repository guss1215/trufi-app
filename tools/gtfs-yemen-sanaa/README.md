# gtfs-yemen-sanaa

Generates the Sana'a GTFS feed using [`trufi-association/trufi-gtfs-builder`](https://github.com/trufi-association/trufi-gtfs-builder) as a Node library — no fork, no submodule. The pinned version lives in [`package.json`](package.json).

## Run

```bash
npm install --legacy-peer-deps   # the builder pulls in conflicting peer deps
npm start
cp out/sanaa.gtfs.zip ../../assets/routing/sanaa.gtfs.zip
```

Output: `./out/sanaa.gtfs.zip`. Other files under `out/` are also written but only the zip is the canonical artifact.

## Input

Reads `../pbf-yemen-sanaa/out/sanaa.osm.pbf` (generated by the sibling tool). If it isn't there, run that tool first. (The Overpass mode is wired up but unreliable; use the PBF.)

## Sana'a tweaks

City-specific values live in [`index.ts`](index.ts): bounding box, `transformTypes`, agency / timezone (`Asia/Aden`) / calendar / headway / vehicle speed / feed metadata, and the `fakeStops` synthesis used because Sana'a's bus relations have no stops mapped in OSM.

## About the committed feed

`out/sanaa.gtfs.zip` is generated by this tool from current OSM: 4 bus lines
(refs 7, 14, 24, 7/14) across 194 directional/variant trips, all-day 5-minute
frequency service. It supersedes the earlier hand-supplied feed and is the one
bundled in the app.

`skipRoute` is inverted vs its name (see the note in [`index.ts`](index.ts)):
the builder keeps a route when it returns `true`. Returning `false` for all —
the obvious-looking "don't skip anything" — silently drops every route and
yields an empty feed. Returning `true` keeps them.

Regenerating from *current* OSM may shift coverage as the map evolves — re-run
and re-test routing in the app.

## Pin

`package.json` pins `trufi-gtfs-builder` to release `v2.13.4` via a GitHub tarball URL. Bump that ref to upgrade.
